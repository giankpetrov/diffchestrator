## 2023-11-20 - [CRITICAL] Terminal Command Injection via `terminal.sendText`
**Vulnerability:** Shell command injection across multiple VS Code commands (e.g. `openClaude`, `aiCommit`, `askClaude`) because user-controlled or external strings (like repository paths and Git diff contents) were interpolated directly into terminal commands using `terminal.sendText` without proper shell escaping.
**Learning:** `vscode.window.createTerminal().sendText(cmd)` passes strings directly to the user's default shell (e.g. bash, PowerShell, cmd.exe). Relying on simple string replacement like `replace(/'/g, "'\\''")` or blindly quoting strings (`"${path}"`) is ineffective and dangerous across different OS environments, making it possible for paths or code diffs to execute arbitrary commands if they contain shell metacharacters like `;`, `$()`, or `|`.
**Prevention:** Use a dedicated shell escaping utility that checks `vscode.env.shell` to apply context-aware escaping. For POSIX shells use single quotes `''` escaping `\'`; for PowerShell use single quotes `''` escaping `''`; for CMD use double quotes escaping `""`. Always pass dynamic arguments through this utility before invoking `terminal.sendText`.
